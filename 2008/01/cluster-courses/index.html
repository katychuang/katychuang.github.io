Cluster Analysis on Course Descriptions

Posted on January 8, 2008 with labels: pydata

Introduction

A study on course descriptions can reveal a general overview of topics taught in a particular graduate program (i.e. Drexel’s iSchool) or even track educational trends. Unfortunately, the process of manually collecting a large dataset can be tedious. As a solution, information extraction can accelerate data collection by reading unstructured text and converting to database form.

For this class project, I would like to find out if Drexel IST courses can be separated into distinct subject categories. The themes may not be apparent to someone who wants an overview of number of courses per specialty. This project incorporates some simple information extraction techniques for collecting data. In addition, hierarchical cluster analysis will be performed to identify the major sub-groups.

The results of this analysis can be potentially useful for those interested in the education of information professionals, namely: educators interested in educational trends and potential students applying to programs. The results may also be useful for evaluating the wording of each course (i.e. some descriptions may be too vague). A brief literature review shows that there exist previous projects and published research on information extraction .

Definitions
  • Information extraction: The process of filling the fields and records of a database from unstructured or loosely formatted text.
  • Content analysis: This is used to determine the presence of certain words or concepts within text. It quantifies the concepts based on existence or frequency of terms.
  • Hierarchical Cluster Analysis: Classifying of objects into different groups (clusters), so that the data in each subset (ideally) share some common. Hierarchical algorithms find successive clusters using previously established clusters, whereas partitional algorithms determine all clusters at once.

Approach

The research question for this project is as follows: What are the major categories of courses provided by the iSchool at Drexel University? The answer to this question may be answered by using the following two step approach: (1) data collection using information extraction, (2) analysis of word frequencies using content analysis, and (3) analysis of course similarity based on word occurrences using cluster analysis. This approach was selected because it fits the type of problem presented. The findings from analysis will be described in the next section.

Step 1 Data Collection

I wrote a JAVA program using the porter stemming algorithm to read a web page listing course descriptions . The output of this program segmented the text to find the course numbers, course names and course descriptions of those offered by Drexel IST. Each of the words in the course description was stemmed using a stemming algorithm prior to building the dictionary of variables (unique terms). Stop words such as (and, of, the, on, new, etc) were excluded.

Step 2 Analysis of Word Frequencies

I wrote a second JAVA program to tally the frequencies of words in a matrix format (figure 1). The dictionary from before was incorporated into this process of breaking text into tokens and seeing if they were in the dictionary. Each word in the dictionary represents a column in the matrix shown below. The disadvantage to using words instead of phrases is some information loss on concepts. For instance, “information systems” have a different meaning from “information”. The advantage of using a JAVA program is the matrix of word frequencies was constructed quickly.

Figure1. Matrix of Word Occurrences

Step 3 Analysis of Course Similarities

Once the matrix was imported into SPSS, hierarchical cluster analysis was selected with the following settings of: complete linkage (furthest neighbor) method, Pearson correlation measure, and no standardization of values. Those particular settings are used because they generated a dendrogram with fairly even clusters (see appendix).

Results

What I learned from this project on creating a web crawler to extract web information barely scratched the surface of this research area. I encountered several challenges with creating the JAVA program because of my lack of skills and knowledge about particular information extraction algorithms. This leads me to the conclusion that it would be very useful to have a text mining background. Results from each step are described below.

Data Collection

There are 55 ALA-approved LIS programs, each with a different format of displaying course information. Thus, this project only looks at Drexel graduate iSchool courses. There were a total of 86 master and doctoral courses in this program; however, INFO 998 Dissertation Study was not included in the data analysis because it is assumed that such an independent study would be fundamentally different from lecture classes. For each course, information about the course number, course title and description were collected.

Content Analysis

685 variables were found based on words that appear in all the course descriptions, but excluding stop words. The frequencies of how many times each word appeared in a course description was recorded and presented in matrix form (figure 1).

Cluster Analysis

Hierarchical cluster analysis was done on 85 master and doctoral level courses. The result showed 6 major clusters: Basic courses, Advanced Courses, Special Librarianship, Information Systems, HCI, and Software Engineering. The appendix contains a list of the courses in each cluster. It is apparent that some of these clusters can be separated into subgroups, but that was not done for this analysis.

Discussion

This project was a way to practice multivariate analysis on a large dataset as well as try my hand at creating a web crawler. In the beginning, the tentative research question began as, “What are ALA-approved LIS program specialties based on an analysis of course descriptions?” However, since the parsing step was very difficult, the project was scaled down to just analyzing Drexel’s courses.

If the analysis was done between multiple library and information science programs, you would be able to see that some programs are more similar than other programs based on material taught. For instance, two schools known for cataloging and archives would be a lot closer together in similarity than the third on informatics. However, this analysis was done on one school, so we can look for clusters within the course offerings.

It turns out that the 6 clusters in the results correspond fairly close to the specializations offered in this program. The specializations in the library and information science program include Digital Libraries, Youth Services, Library & Information Services, School Library Media Program, and Competitive Intelligence. Drexel also offers degrees in Software Engineering and Information Systems. The chart in the appendix shows courses from each cluster that matches with specializations.

There are many web mining applications using both cluster analysis and information extraction. In an interview, two leading researchers mention that “Cluster analysis is an important tool for information extraction in data mining. It groups similar objects together according to their attributes or behavior.” Other domains where this may be useful include job search (flipdog.com), information about people (zoominfo.com), citation information (citeseer.org), medical insurance claim forms (verity.com) and email messages .

Further research can be done using multidimensional scaling to visualize relationships among courses, and possibly help students and their advisors select appropriate courses. It may be a good idea to include additional information about requirements and term offerings in the information extraction process.
