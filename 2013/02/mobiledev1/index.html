Constraints in Mobile Development, pt 1

Posted on February 6, 2013 with labels: tutorial

There are different IDE’s for developing web apps, I’ve been thinking about how the environment and documentation affects development. I’ve worked with a number of tools over the years for various languages and aspects of web development*. Some tools are better suited for backend tasks and others for front end. Regardless, there are some common patterns in software development workflow.

Some context for non-developers here.. IDEs are integrated development editors, software that helps you write code. It could be very simple (just a text editor) or have all the bells and whistles (drag and drop, design view and code view, etc). Some come with the OS (vim) and others require purchasing an expensive license to install (m$). For the sake of this blog essay, I’m more interested in comparing android and ios mobile development, and at a macro level, since I want to focus more on the development process. See figure 1 & 2 for screenshots.

IDE INTERFACES OVERVIEW

Android Development Toolkit (ADT)
They have free tools for developers, the popular one is Eclipse with ADT plugin. Big download, tricky to set up the first time on mac. > ADT extends the capabilities of Eclipse to let you quickly set up new Android projects, create an application UI, add packages based on the Android Framework API, debug your applications using the Android SDK tools, and even export signed (or unsigned) .apk files in order to distribute your application.

Figure 1. Eclipse with ADT

Xcode

Apple will let you download XCode with the iPad/iPhone simulator for free if you sign up for a developer key. Once you have that installed, you open it up and can start creating projects and coding. No additional configurations needed to start coding.
